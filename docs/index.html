Sobre el proyecto...

El proyecto online-shopping es una aplicación de comercio electrónico desarrollada utilizando tecnologías como Spring Boot, Spring Web Flow, Spring Rest Services y Hibernate. También incluye Spring Security con configuración en Java y anotaciones.

Code smells

El proyecto presenta varios problemas que afectan su calidad y mantenimiento. Se observan métodos largos en clases de servicios, lo que dificulta la comprensión del código. Hay duplicación de código, especialmente en la lógica de validación, lo que genera inconsistencia. Algunas clases actúan como intermediarios innecesarios, señalando un posible acoplamiento excesivo. El uso de tipos primitivos para conceptos clave reduce la expresividad del código, y los nombres de métodos y variables son a menudo poco claros. Además, la falta de pruebas unitarias compromete la estabilidad y confiabilidad del sistema.

Clean code

El proyecto tiene problemas que violan principios de código limpio, como YAGNI, KISS y DRY. Métodos en AddressServiceImpl parecen innecesarios, mientras que el filtrado de categorías en CategoryServiceImpl podría simplificarse para evitar lógica extra. En ProductServiceImpl, la duplicación de código para filtrar productos inactivos se puede reducir consolidando esta lógica en un solo lugar. Además, se rompen principios de diseño clave: UserServiceImpl viola el principio de Responsabilidad Única (SRP) al manejar tanto la codificación de contraseñas como la gestión de usuarios, y las clases de servicio están fuertemente acopladas a las implementaciones de repositorio, incumpliendo el principio de Inversión de Dependencia (DIP).

Abordando Deuda Técnica

Se añadieron pruebas unitarias para mejorar la calidad y funcionamiento del proyecto de compras en línea. Antes, solo había una prueba documentada, pero ahora hay un conjunto que verifica las clases principales del paquete com.isolutions4u.onlineshopping.model. Estas pruebas aseguran que elementos clave como Address, CartLine, Cart, Category, Product, RegisterModel, UserModel y User estén funcionando correctamente. Para ejecutar las pruebas, se usa el comando mvn test en el directorio del proyecto. Se realizaron 50 pruebas, lo que mejora la confiabilidad y estabilidad del sistema, contribuyendo a una mejor experiencia para los usuarios.

Análisis con Herramienta SonarCloud

SonarCloud es un servicio en la nube que analiza el código fuente para identificar problemas relacionados con calidad, seguridad y mantenibilidad. Ofrece herramientas para detectar errores, código duplicado, complejidad innecesaria y vulnerabilidades, proporcionando métricas y sugerencias para mejorar el código. Se integra con plataformas como GitHub, GitLab y Bitbucket para automatizar la revisión de código con cada cambio, facilitando el proceso de integración continua. SonarCloud ayuda a los equipos de desarrollo a mantener altos estándares de calidad, ofreciendo informes detallados y métricas que permiten rastrear la evolución del código y abordar problemas de manera proactiva. Esto fomenta una cultura de mejora continua y contribuye a la creación de software más seguro y confiable.

Desarrollo de Experiencia (DevEx) en el Proyecto

El Desarrollo de Experiencia (DevEx) busca mejorar la productividad y satisfacción de los desarrolladores. En el proyecto de refactorización de compras en línea, se aplicaron pruebas unitarias, refactorización y SonarQube, lo que mejoró la calidad y mantenibilidad del código, redujo la deuda técnica y aumentó la confiabilidad del sistema. Sin embargo, hay áreas que necesitan mayor aplicación de principios SOLID, como responsabilidad única e inversión de dependencia. Además, la introducción de herramientas como SonarQube puede añadir complejidad al proceso de desarrollo. A pesar de estos desafíos, las mejoras realizadas han tenido un impacto positivo en la experiencia del desarrollador.

SpaceFramework

El modelo SPACE evalúa cinco áreas clave para mejorar la productividad y bienestar en equipos de desarrollo: satisfacción y bienestar, rendimiento, actividad, comunicación y colaboración, y eficiencia y flujo. En el proyecto de Refactorización de Compras en Línea, aplicar el modelo SPACE ayudó a mejorar la calidad del código y a reducir la deuda técnica, promoviendo un entorno de trabajo más satisfactorio. El rendimiento aumentó con pruebas unitarias y análisis estático, permitiendo correcciones más rápidas. La comunicación y colaboración mejoraron con la integración continua y revisiones de código. La eficiencia y flujo se incrementaron con la refactorización continua y la automatización de tareas. Estas mejoras crean un entorno más productivo y colaborativo para el equipo de desarrollo.

Análisis de Seguridad CodeQL

CodeQL es una herramienta de análisis estático de código de GitHub que detecta vulnerabilidades de seguridad, errores de codificación y otros problemas en varios lenguajes de programación. Para usarla en GitHub Actions, se configura un archivo de flujo de trabajo (codeql.yml) en el directorio .github/workflows. Este archivo define cuándo y cómo se realiza el análisis, incluyendo la ejecución en ramas específicas y horarios programados. CodeQL puede identificar vulnerabilidades como inyecciones de SQL, desbordamientos de búfer, y problemas de Cross-Site Scripting (XSS), además de errores de codificación como uso incorrecto de funciones, variables no inicializadas y errores de manejo de excepciones. También ofrece sugerencias para mejorar la calidad del código, abordando problemas como código redundante, complejidad ciclomática alta y violaciones de buenas prácticas de codificación.
